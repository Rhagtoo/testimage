# Discovery Engine для pentest-таргета — нерешённые вопросы

Строим discovery engine для поиска галерей на pentest-таргете. Цель: найти максимум публичных галерей, используя ID-пространство и API.

## Контекст (что уже знаем)

**ID: 7 символов base62** (62^7 ≈ 3.5×10^12 возможных). Выглядят случайными.
**Кластеризация есть только внутри сессии** — 34% ID имеют соседей по первым 5 символам в рамках одного browser-сеанса. Между сессиями — 4%. Похоже на `session_seed + counter`.
**Галереи живут ~30 минут** — потом 404. Быстрая деградация.
**JSON API доступен** через сессионную куку, извлечённую из браузера. До 100 rps из WSL.
**Запись (upload) — только через реальный браузер**, Cloudflare блокирует автоматические запросы. Через WSL + полные браузерные заголовки можно добавлять контент, но не создавать.
**CF Worker + VPN-ротатор** = 2-3 rps, 80 IP-точек для обхода банов.
**Старые кластеры мертвы** — данные месячной давности невалидны.
**Публичных листингов** (explore/recent/browse) на сайте нет.

## Ключевые вопросы

### A. Как найти чужие объекты?

1. Есть ли скрытый endpoint для получения списка ВСЕХ объектов (не только своих)? Возможно admin/superuser API?
2. Как эффективно скрапить публичные ссылки из внешних источников — поисковики, форумы, code-репозитории, соцсети? Какие дорки/запросы вы бы использовали?
3. Есть ли у таких сервисов sitemap'ы, RSS-фиды, или другие механизмы раскрытия URL?
4. Можно ли зарегистрироваться в официальном API сервиса (api.*) и получить доступ к search/discovery эндпоинтам?
5. Какие ещё есть способы найти случайные URL хостинг-сервисов, кроме прямого перебора?

### B. Как устроена генерация ID?

1. Если ID = `HMAC(session_secret, counter)` — бесполезно для поиска чужих. Но если это `timestamp + random` или `LCG/MT19937` — можно предсказывать. Как отличить одно от другого, имея выборку ID?
2. Какие методы reverse-engineering PRNG вы бы применили к 7-символьным base62 ID? Нужен ли >1000 сэмплов из одной сессии?
3. Если ID содержит временную метку (скрытый timestamp в нескольких символах) — как это детектировать статистически?

### C. Как масштабировать опрос API?

1. Стабильно держим ~100 rps через извлечённую сессионную куку, но через 1-2 минуты начинаются 500-ошибки. Это rate limiting или естественная деградация сессии? Как продлить?
2. CF Worker прокси банится волнами по IP-диапазонам Cloudflare. Можно ли маршрутизировать Workers через не-Cloudflare IP?
3. Какие стратегии обхода rate-limiting вы знаете для подобных API без аутентификации?

### D. Альтернативные векторы атаки

1. Кроме галерей, есть CDN-домен с файлами (`cdn.target/XXXX/filename.jpg`) — имеет ли смысл перебирать файловые ID вместо галерейных?
2. Есть ли смысл искать через reverse image search (Google Images, TinEye, Yandex) — индексируют ли они такие сервисы?
3. Wayback Machine / archive.org / CommonCrawl — можно ли восстановить старые ID из кэшей?
4. Если сайт недавно переехал на новый домен — есть ли на старом домене данные, которых нет на новом?

**Приоритет для решения:**

2. Как скрапить URL сервиса из поисковиков?
3. Как стабилизировать доступ на >10 минут без бана?
